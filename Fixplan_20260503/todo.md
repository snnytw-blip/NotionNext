## 役割分担の考え方

| 役割 | モデル | やること |
|------|--------|----------|
| **設計レビュアー** | DeepSeek V3（Pro） | コードを読み・問題を特定・修正仕様を生成 |
| **実装者** | DeepSeek V3 Flash | 仕様書通りにコードを書く・編集する |

ProにFlashへの「作業指示書」を出力させるのが核心です。

---

## Step 1: Proへの分析指示（Continue で投げるプロンプト）

対象ファイルをコンテキストに追加した上で、以下を投げます。

```
# コードレビュー依頼

## 目的
以下の改修計画に照らして、現在のコードベースを分析し、
実装担当AIへの作業指示書を生成してほしい。

## 改修計画（P0タスク）
- publish_date を Notion DB から取得してソートに使う
- ソートキーを last_edited_time → publish_date DESC に変更
- category / site プロパティの取得・フィルタ処理を追加
- canonical タグを各記事ページの <head> に自動出力
- favicon をデフォルトから変更
- footer をデフォルトから「© 2026 未来AIラボ」に変更

## 依頼内容
上記タスクごとに、以下の形式で作業指示書を出力してほしい。

---
### タスク名
**対象ファイル:** `path/to/file.js`
**現状コード（該当箇所）:**
\`\`\`js
// 現在こうなっている
\`\`\`
**問題点:** なぜこれが問題か
**修正仕様:**
- 変更すべき内容を箇条書きで
- 追加すべきロジックを明記
- 変数名・プロパティ名も具体的に

**実装担当への注意事項:** 壊してはいけない周辺ロジック等
---

## 制約
- 推測で書かない。コードが見つからない場合は「未確認」と明記
- ファイルパスは実在するものだけ書く
- 一度に全部やらず、P0の6タスクに絞る
```

---

## Step 2: Proの出力イメージ（こういう形で返ってくる）

```
---
### publish_date ソート修正

**対象ファイル:** `lib/notion.js`

**現状コード:**
\`\`\`js
sorts: [{ timestamp: 'last_edited_time', direction: 'descending' }]
\`\`\`

**問題点:** last_edited_time は編集するたびに変わるため
          公開順が崩れる

**修正仕様:**
- sorts を publish_date プロパティ基準に変更
- Notion DB に publish_date (Date型) が存在することを前提
- プロパティ名: `publish_date`

**実装担当への注意事項:**
- getDatabase() を呼んでいる箇所はここだけか確認
- フォールバックとして created_time を残すか検討
---
```

## Step 3: Flashへの実装指示

Proが出力した作業指示書を**そのままコンテキストに貼って**、Flashに投げます。

```
# 実装依頼

以下の作業指示書に従って、コードを修正してください。

## ルール
- 指示書に書かれたファイル・箇所だけを変更する
- 指示書にない箇所は触らない
- 変更した箇所にはコメントで `// [publish_date対応]` のように
  タスク名を残す
- 変更後のコード全体を出力する（diffではなく）

## 作業指示書

```

---

## 運用上のポイント

**タスクは1つずつ渡す**
Flashに複数タスクを一度に渡すと干渉しやすい。指示書を1タスク単位に分割して順番に投げる。

**Proのレビューを先に全部終わらせる**
コードを読むのはProだけ。Flashにはコードを読ませず「書くだけ」に専念させると精度が上がる。

**Flashの出力をProに検証させる（任意）**
実装後に `このコードで意図通りに動くか確認して` とProに渡せば、ループ式のレビューサイクルになる。

## モデル切り替えの操作（Continueの場合）

Continueのチャット欄でモデルを切り替えるか、`config.json` に両モデルを登録しておいて会話ごとに選択します。

```json
// config.json の models に両方登録しておく例
{ "model": "deepseek/deepseek-chat", "title": "DeepSeek V3 Pro" },
{ "model": "deepseek/deepseek-v3-flash", "title": "DeepSeek V3 Flash" }
```
